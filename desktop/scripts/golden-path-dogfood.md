# Week 1 Golden Path — founder dogfood protocol (Horizon 1 exit)

## Goal

Answer one question with evidence:

**“Week 1’i yardım almadan bitirebildin mi?”**

Matrix geçmek ≠ kullanıcı ship eder. This protocol validates the **founder golden path**, not CI.

## Cohort

- **3–5 technical founders** (can open a local repo, run npm, tolerate rough edges)
- Mix of thesis types if possible: landing conversion, viral short-form, SEO — at least 2 mechanisms
- Facilitator **silent** unless blocked >10 minutes

## Setup (each session)

1. Fresh project folder (real landing or SaaS repo — not empty)
2. Backend connected; Computer Use enabled when testing Lane A verify
3. Screen record full session (Loom/OBS)
4. No facilitator hints on “where to click” after minute 5

## Golden path script (founder self-serve)

| Step | Founder does | Pass criterion |
|------|----------------|----------------|
| 1 | Open project → intake completes | Channel thesis visible |
| 2 | Seal strategic decision | Ops Week 1 table appears |
| 3 | Start first **system** ops task | Edit run produces diff |
| 4 | **Run validation** → Apply | Typecheck/lint gate respected (or explicit override with reason noted) |
| 5 | Browser verify (if CU online) | Task not marked done until verify passes OR clear “CU offline” handoff |
| 6 | Complete first **human** task | Kit opens → URL/KPI proof submitted |
| 7 | Reload app (quit + reopen project) | Ops cadence, kernel state, proof intact (Faz 10) |
| 8 | Finish remaining Week 1 ops | All terminal or honestly skipped |
| 9 | Start Week 2 (`Start Week 2`) | No mandatory essay; cycle archives silently |
| 10 | Exit interview (1 question) | Score 1–5: “Week 1’i yardım almadan bitirebildin mi?” |

## Exit criteria (Horizon 1)

- **≥ 3/5** founders score **≥ 4** on the exit question
- **≥ 4/5** complete Steps 1–6 without facilitator rescue
- **0/5** report “task marked done but verify never ran” when CU was required
- **0/5** report data loss after reload on ops/kernel/proof

## Anti-patterns (fail the session)

- Facilitator navigates to Plan Studio as “next step” during active ops week
- Week review modal forced before Week 2
- Apply without validation with no override acknowledgment
- System task auto-completes while verify pending

## After dogfood

File issues with: thesis id, step number, screen record timestamp, expected vs actual.

Do **not** expand scope to Parts 21–28 until this protocol passes.

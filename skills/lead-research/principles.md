## Principles

1. **ICP before volume.** A list of 20 perfect-fit accounts beats 200 random names. Define must-have criteria before searching.
2. **Evidence-backed candidates only.** Every account/person needs a public source URL or repo signal — never invent emails, revenue, or headcount.
3. **Why-now is the outreach hook.** Fit without timing produces ignored messages. Document trigger: funding, hiring, launch, stack change, regulation.
4. **Qualification tiers: A / B / C.** A = all must-haves + why-now; B = missing one nice-to-have; C = speculative — exclude C from default outreach.
5. **Source matrix prevents double-counting.** Track where each lead came from (LinkedIn search, PH makers, community, inbound) and dedupe by domain.
6. **Respect permission boundaries.** Public profiles and company sites only — no login-walled scraping, no purchased lists presented as researched.
7. **Scorecard beats gut feel.** Use explicit fit dimensions (role, stack, pain, budget, timing) scored 1–5 with notes.
8. **Sales persona ≠ spam persona.** Research output is for the founder to review — drafts come later in `outreach-drafting`.
9. **Cap at 20 default candidates.** Depth on 20 beats shallow 100; user can ask for expansion after review.
10. **Tie every lead to product intelligence.** If the product profile is weak, pause and run `product-intelligence` first.

# Maker's First Comment — Templates

The maker's first comment is pinned to the top of the discussion. After the tagline, it's the most-read text on the page. It should sound like a person, not a brand. 150–250 words. Plain English. No bullet lists if you can avoid them — prose feels human.

---

## Template 1 — The "Why we built it" story (default; use this 80% of the time)

```
Hey Product Hunt 👋

I'm [name], one of the makers of [product]. Quick story on why it exists:

For the last [N] years I was [role] at [company / type of company], and every [time period] I'd run into the same problem: [specific painful scenario in one sentence]. We tried [thing 1], we tried [thing 2], we hacked together [thing 3]. Nothing solved it properly.

So in [month/year], [co-founder name] and I started building [product]. The core idea is simple: [one-sentence mechanism]. In practice that means [concrete outcome the user sees].

We've been in private beta with [N] teams including [2–3 named users if possible]. They've helped us cut [metric] from [X] to [Y].

Today's launch is the public version. It's free to try (no card), and [pricing model] when you're ready for more.

What I'd genuinely love from this community:
- If you've felt this pain — tell me your version of the story. We may not have solved your exact case yet.
- If something in the demo doesn't make sense, ask. I'll be here all day.
- And of course, if it helps you, we'd be honored if you'd share it with one person who'd find it useful.

Thanks for reading — going to refresh this tab way too many times today.

— [name]
```

## Template 2 — The "Show, don't tell" comment (use when the product is obvious from the visual)

```
👋 Maker here.

The 90-second demo at the top covers what [product] does, so I'll skip the pitch.

Three things I want to be upfront about:

1. **What it's great at today:** [specific 1-line use case], [specific 1-line use case].

2. **What it's not for (yet):** [honest gap]. We're working on it but [time frame].

3. **What I'm hoping to learn here:** whether [specific question that genuinely matters to your roadmap].

Try it free at [link] — no email required for the demo, only for save. If you want a 1:1 walkthrough, drop a comment and I'll DM you a Calendly.

Going to be in the comments all day. Bring your hardest use case.

— [name]
```

## Template 3 — The relaunch / v2 comment

```
👋 Some of you remember our first launch on [month, year]. We hit [rank] and got some incredible feedback — and a clear message: [the criticism, stated honestly].

So for the last [N] months, we rebuilt [thing]. Specifically:

- [Concrete change #1 with a metric]
- [Concrete change #2]
- [Concrete change #3]

The product today is [headline differentiator] without [the pain people complained about last time].

If you tried us before and bounced — I'd love for you to take another look. If you're new — welcome. I'll be here answering comments until late tonight Pacific.

— [name]
```

---

## Reject the comment if it:

- Starts with "We're excited to announce…"
- Lists features as bullet points instead of telling a story
- Contains the words "leverage," "synergy," "revolutionary," "game-changer"
- Doesn't include the maker's first name
- Asks for upvotes directly
- Is longer than 350 words (people stop reading)
- Reads like marketing copy a content team wrote — PH readers can smell it from the first sentence

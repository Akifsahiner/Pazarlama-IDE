# SEO Content Engine

Professional SEO for dev founders: intent-mapped clusters, template-driven comparison pages, and honest indexing baselines — not "improve SEO" fluff.

Use when `scan.routes` includes `/blog`, `/docs`, or marketing pages beyond a single LP.

## Process
1. Baseline GSC + indexation (`seo_gsc_baseline_audit`).
2. Map keyword clusters by intent tier (`seo_intent_cluster_map`).
3. Ship comparison / alternatives / vs pages from templates (`seo_comparison_page_template`, `seo_alternatives_page_template`).
4. Wire internal link graph docs ↔ blog ↔ product (`seo_internal_link_graph`).
5. Place docs/blog CTAs without breaking reader trust (`seo_docs_blog_cta_placement`).
6. Refresh underperformers at D+30 (`seo_content_refresh_d30`).

## Output
- Keyword cluster sheet with priority URLs
- 3–5 indexable page briefs (comparison, alternatives, how-to hub)
- Internal link map (hub → spoke → signup)
- Kill rules when impressions stall or thin content risk appears

## Ethics
- No fake traffic, no keyword stuffing, no AI slop pages without product-specific proof
- Programmatic pages must pass "would I link this to a customer?" test

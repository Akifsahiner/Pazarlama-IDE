# Playbook: B2B Dev Tool SEO

Developer tools win on **docs depth + commercial comparison** — not lifestyle blog posts. ICP reads docs, GitHub README, and "[Competitor] alternative" before buying.

- **Docs as SEO surface:** Every integration guide links to vs page + signup.
- **Open-source:** README "Compare to X" links to canonical on-site comparison (not duplicate README essay).
- **Comparison tables:** Self-host vs cloud, API limits, pricing unit — dev buyers filter on these rows.
- **Programmatic guardrails:** `[Competitor] alternative for [language/framework]` only when you have real integration story.

## Preconditions
- [ ] `business_model` is saas, tool, open_source, or developer_tool
- [ ] Technical reviewer assigned for comparison accuracy
- [ ] API/docs route indexable

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | API docs only | 1 vs page for primary competitor |
| standard | Docs + changelog | Alternatives + 2 vs + how-to hub |
| aggressive | Category leader challenger | 5-page cluster + programmatic framework pages |

## Timeline
- **T-21**: SERP teardown of competitor docs SEO
- **T-14**: Alternatives page with pricing/API rows
- **T-10**: vs pages for #1 and #2 competitor
- **T-7**: How-to hub "How to [job] with [Product] vs [legacy approach]"
- **T-3**: Schema + docs CTA component

## Orchestration
- Primary: docs + comparison
- Parallel: GitHub README sync (same URL, not duplicate content)
- Do not: G2/Capterra scrape as only source — verify primary docs

## Realistic outcomes
- Niche dev tool: rank page 1–3 for "[competitor] alternative" in 90–180d
- Enterprise-only ICP → lower volume, higher signup quality

## Kill / pivot rules
- Technical inaccuracy called out in comments → hotfix within 24h
- Comparison page ranks but dev signup <1% → developer onboarding gap, not SEO

## Ethics line
- Open-source: disclose commercial vs OSS edition honestly
- No FUD on competitor security without cited CVE/advisory

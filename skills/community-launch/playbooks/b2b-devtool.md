# Playbook: B2B DevTool Community

Your buyer is an engineer or tech lead. HN and specific subreddits (r/devops, r/webdev, r/SaaS) beat broad consumer channels.

- **Position as workflow fix, not "platform".** Show HN title names the job: "Show HN: CI cache for monorepos – 40% faster builds."
- **Open-source angle:** If repo is public, link GitHub in first comment; star CTA in README not title.
- **Proof:** One real metric from design partner (even n=3) beats vanity claims.
- **Sales handoff:** Inbound demo spike expected H0–D+3; SDR script ready for "saw you on HN."
- **Skip consumer subreddits** (r/Entrepreneur spam tone) unless founder story is genuinely educational.

## Preconditions
- [ ] ICP = developers / technical buyers
- [ ] Demo or OSS repo public
- [ ] Landing message match for technical reader

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | Stealth B2B | IH + niche sub only |
| standard | Dev tool ready | Show HN + IH |
| aggressive | OSS + design partners | Full blitz + Dev.to deep-dive |

## Timeline
- **T-14**: Technical community map (HN + 2 subs)
- **T-7**: Demo + docs link QA
- **H0**: Show HN
- **H+72**: Dev.to architecture post
- **D+1**: Teardown with pipeline notes

## Orchestration
- HN primary; Dev.to for long-tail; LinkedIn teardown for enterprise inbound
- PH optional separate week — not same 72h

## Realistic outcomes
- Strong dev tool + OSS: 200–500 points possible; enterprise inbound lumpy
- Weak demo: engineers roast in comments — fix before post

## Kill / pivot rules
- Technical objections unanswered → pause marketing; ship fix post
- ICP mismatch comments → narrow positioning before retry

## Ethics line
- No astroturfing in GitHub issues or fake "user" comments

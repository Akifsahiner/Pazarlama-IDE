# Niche Dev Newsletter Sponsorship

Professional sponsorship runbook: shortlist niche newsletters, score audience overlap, ship native ≤120-word creative, UTM attribution, and CPA kill rules.

Use when ICP reads dev newsletters and you have a **working landing page + UTM analytics** — not vaporware or enterprise-only buyers.

## Process
1. Shortlist 8–12 niche newsletters (`nl_sponsor_shortlist_niche`).
2. Score audience overlap vs ICP (`nl_audience_overlap_score`).
3. Request media kit + last 3 sponsor examples (`nl_media_kit_request`).
4. Draft sponsor brief + native copy (`nl_sponsor_creative_brief`, `nl_native_copy_120_words`).
5. Lock UTM scheme before booking (`nl_utm_campaign_scheme`).
6. Book slots with ≥14d lead time (`nl_slot_booking_lead_time`).
7. A/B hook on first two sends (`nl_creative_ab_hook`).
8. Kill if 7d CPA exceeds ceiling (`nl_kill_rule_cpa_7d`).
9. Teardown D+7 (`nl_post_sponsor_teardown`).
10. Max 2 consecutive slots per newsletter (`nl_max_consecutive_slots_2`).

## Output
- Sponsor shortlist with overlap scores + slot pricing
- Paste-ready native copy + UTM-tagged URLs
- Kill rules when CPA or CTR misses after 7d

## Ethics
- No fake endorsements, undisclosed affiliate framing, or misleading "editor's pick" claims
- Native tone — reads like editorial tip, not banner ad

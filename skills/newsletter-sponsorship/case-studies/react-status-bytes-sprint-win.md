# Case study: React Status + Bytes sprint → 142 signups, $118 CPA (anonymous)

## Context
B2B dev SaaS (CI observability), $4.2k sprint budget, free tier + 8 design partners. No paid social. LP with sandbox signup <45s.

## Tactic stack applied
1. `nl_sponsor_shortlist_niche` — 11 newsletters; top 3 overlap 4.5/5
2. `nl_media_kit_request` — React Status 42% open, Bytes sponsor CTR ~1.1% cited
3. `nl_native_copy_120_words` — Hook A outcome / Hook B pain
4. `nl_utm_campaign_scheme` — `nl_sprint_202603`, distinct `hook_a` / `hook_b`
5. `nl_kill_rule_cpa_7d` — ceiling $200 signup; no kill triggered
6. `nl_post_sponsor_teardown` — D+7 per slot

## Sends
| Issue | Cost | Clicks | Signups (7d) | CPA |
|-------|------|--------|--------------|-----|
| React Status (Hook A) | $1,800 | 410 | 68 | $132 |
| Bytes (Hook B) | $1,400 | 520 | 54 | $148 |
| JS Weekly (Hook A) | $1,000 | 280 | 20 | $200 |

## Outcome
- 28d sprint: 142 signups, blended CPA $118 (weighted)
- 14d activation: 31% started first CI integration
- 30d: 19 paid from newsletter cohort ($4.8k ARR)

## Lesson
Hook B won on Bytes (1.3% CTR vs 0.9% on React Status Hook A) but React Status drove higher-quality team domains. Teardown separated *volume* from *pipeline* — scaled React Status 2 more slots in Q2, skipped JS Weekly rebook.

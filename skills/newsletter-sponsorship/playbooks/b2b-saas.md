# Playbook: B2B Dev SaaS

Your buyer is an engineer, platform lead, or technical founder. Sponsor newsletters where they **already read weekly** — React Status, Bytes, Node Weekly, DevOps Bulletin, Pragmatic Engineer (select issues), TLDR Dev — not generic business digests.

- **Position as workflow fix in native copy.** "Cut CI time 40% on monorepos" beats "all-in-one dev platform."
- **Link to technical eval path.** Free tier, OSS repo, or docs quickstart — not "book a demo" as primary CTA unless ACV >$20k.
- **Proof from design partners.** One metric from 3–10 logos (even anonymized) beats feature bullets.
- **Sales handoff:** Expect inbound spike H0–D+3; SDR script: "Saw us in [Newsletter] — what stack are you on?"
- **Skip consumer newsletters** (general productivity, crypto, marketing) even if cheap CPM.

## Preconditions
- [ ] ICP = developers / technical buyers (ICP doc cites stack)
- [ ] LP speaks to technical reader (code sample, architecture diagram, or live sandbox)
- [ ] CRM or inbox tag for `utm_source` newsletter referrals

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | Early B2B, no proof | 1 slot; learning CPA only |
| standard | Free tier + 3 design partners | 2–3 niche newsletters over 6 weeks |
| aggressive | Clear PMF + $3k/mo | `aggressive-sponsor-sprint` |

## Timeline
- **T-21**: Map newsletters to stack (React → React Status, JS → Bytes, etc.)
- **T-14**: Media kits + sponsor block screenshots from last 3 issues
- **T-10**: Technical native copy + brief
- **T-7**: UTM per newsletter; sales alert configured
- **H0**: First send (highest overlap list)
- **H+7**: Second send (different stack newsletter if multi-stack ICP)
- **D+7 per send**: Teardown with activation rate, not just signups

## Tactic stack
1. **`nl_sponsor_shortlist_niche`** — Stack-aligned titles only
2. **`nl_audience_overlap_score`** — Require ≥4/5 for B2B dev
3. **`nl_media_kit_request`** — Ask B2B sponsor history (devtools, infra)
4. **`nl_sponsor_creative_brief`** — Technical eval CTA, screenshot spec
5. **`nl_native_copy_120_words`** — Outcome-first hook
6. **`nl_utm_campaign_scheme`** — `nl_[newsletter_slug]_YYYYMM`
7. **`nl_slot_booking_lead_time`** — Avoid conference/issue clash weeks
8. **`nl_kill_rule_cpa_7d`** — B2B ceiling typically $150–300 signup
9. **`nl_max_consecutive_slots_2`** — Rotate between React/JS/Infra lists
10. **`nl_post_sponsor_teardown`** — Pipeline notes for sales

## Orchestration
- Primary: highest overlap dev newsletter
- Support: founder LinkedIn teardown post D+7 (not click-bait)
- Do not run PH/HN same week as first sponsor slot

## Realistic outcomes
- Strong dev tool + React/JS fit: 60–150 signups/slot; 20–35% trial activation
- Enterprise-only motion (demo-only LP): high CPC, low signup — wrong channel

## Kill / pivot rules
- Signups are all personal emails with zero team domains → ICP mismatch; pause
- Technical objections in publisher forward emails → fix docs before rebook
- CPA >$400 at D+7 with 200+ clicks → kill newsletter; test LP not list again first

## Ethics line
- No astroturfed "developer love" quotes
- Sponsored label per publisher; no fake "editor's pick"

# Template: Native sponsor copy (≤120 words)

Newsletter blocks should read like the editor sharing a useful tool — not a display ad.

## Structure

1. **Hook (1 sentence)** — concrete outcome or pain
2. **What it does (1–2 sentences)** — workflow, not buzzwords
3. **Proof (optional, 1 sentence)** — honest metric or beta quote
4. **CTA (1 sentence)** — single link, action verb

**Word budget:** 90–120 words. Count before sending brief.

## Voice rules

| Do | Don't |
|----|-------|
| Name the stack (React, K8s, Postgres) | "AI-powered platform" |
| One clear CTA | "Learn more \| Book demo \| Read blog" |
| Honest limitation | Fake social proof |
| Short sentences | Exclamation marks |

## Publisher variants

- **TLDR-style (shorter):** Cut proof sentence; target 75–90 words
- **Long-form newsletter:** May allow 130 words — confirm in media kit; never exceed without OK
- **Logo + screenshot:** Pair with 80-word copy when image carries feature detail

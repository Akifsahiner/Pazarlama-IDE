# Case study: CLI tool ‚Äî 3 awesome-list merges + Show HN ‚Üí 420 stars in 30d (anonymous)

## Context
Open-source Node CLI for monorepo caching. Pre-launch: 12 stars, no registry publish. Founder had 2 weeks of HN comments (no links). Email list: 800 subscribers (38% engaged open rate).

## Repo scan (T-21)
- Added `oss_license_clarity_block`: MIT + "Commercial use permitted" in README
- Shipped `oss_changelog_visible` + v0.1.0 tag with real release notes
- Published `oss_package_registry_listing` to npm with `monorepo`, `ci`, `cache` keywords
- Applied `oss_readme_star_cta` below install block (Template A)

## Distribution stack
1. **`oss_awesome_list_pr`** ‚Äî 4 PRs over 18d; 3 merged (`awesome-nodejs`, `awesome-devops`, niche CI list)
2. **`oss_release_notes_blog`** ‚Äî Dev.to post for v0.2.0 + launch narrative T-3
3. **`oss_show_hn_coordination`** ‚Äî Show HN Tuesday 9:10am PT, 72h after README freeze (`community-launch`)
4. **`oss_github_topics_tags`** ‚Äî `nodejs`, `cli`, `monorepo`, `ci-cd`, `build-tools`
5. **`oss_good_first_issue_label`** ‚Äî 4 issues; 1 external PR merged by D+21

## Maintainer outreach
- **`oss_maintainer_dm_ethics`**: DMs only to 2 merged list maintainers ‚Äî thank-you + "happy to adjust entry wording." Zero cold star asks. No incidents.

## Outcome
| Metric | Result |
|--------|--------|
| Stars (30d) | 12 ‚Üí 420 |
| HN (4h) | 195 points, 41 comments |
| HN sessions (72h) | 3,100 |
| npm downloads (wk1) | 2,400 |
| List PR merge rate | 3/4 (75%) |
| Email wave 1 click‚Üístar | 11% |

## Lesson
Awesome-list traffic compounded *before* HN ‚Äî clones rose 3 weeks pre-HN from list #2 merge. Show HN amplified an already-credible repo; first comment linked npm install + GitHub, not star CTA. Biggest miss: delayed `oss_star_milestone_thread` at 100 stars by 5 days ‚Äî would have captured IH long-tail earlier.

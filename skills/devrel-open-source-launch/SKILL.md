# DevRel Open Source Launch

Repo-first distribution for OSS and dev tools: scan README, CHANGELOG, topics, and registry listings before any community spike.

Use when the product has a **public GitHub repo** with install docs and a clear license block — and you plan Show HN or IH in the same launch window.

## Process
1. Repo scan: README hero, license, topics, good-first-issue (`oss_license_clarity_block`, `oss_github_topics_tags`).
2. Ship README star CTA + visible CHANGELOG (`oss_readme_star_cta`, `oss_changelog_visible`).
3. Queue 3–5 awesome-list PRs with maintainer DM ethics (`oss_awesome_list_pr`, `oss_maintainer_dm_ethics`).
4. Publish release notes as blog/Dev.to distribution (`oss_release_notes_blog`).
5. Coordinate Show HN ≥48h after repo polish (`oss_show_hn_coordination`).

## Output
- T-21 → D+7 OSS launch calendar tied to tactic IDs
- Paste-ready README CTA, awesome-list PR, release notes template
- Kill rules when stars stall or maintainer DMs get flagged

## Ethics
- No star-for-star swaps, no fake contributors, no mass cold DMs without prior relationship
- Awesome-list PRs add value to the list — not SEO spam
- Maintainer DMs: one touch, opt-out respected, never "please star my repo"

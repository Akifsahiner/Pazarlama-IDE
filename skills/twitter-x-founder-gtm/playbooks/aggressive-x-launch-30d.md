# Playbook: Aggressive X Launch 30d

30-day founder X war room: ICP map → build-in-public → launch thread → ethical quote-tweets → D+1 teardown. **Not** LinkedIn reposts or engagement pods.

## Preconditions
- [ ] Founder X account with real name + face/brand avatar
- [ ] Launch date within 30 days (or relaunch window)
- [ ] Demo screenshot or 30s GIF shippable
- [ ] UTM scheme: `utm_source=twitter&utm_medium=organic`
- [ ] Calendar: 30 min/day engagement + 2h launch window blocked
- [ ] No stealth / founder-won't-post constraints

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | <300 followers, new account | Bio + 2 threads/month + daily replies only |
| standard | 300–2k followers | Build-in-public 3×/week + 1 pre-launch thread |
| aggressive | ≥2k followers or warm dev network | Full 30d calendar + launch thread + quote-tweet window |

**Honest ceiling:** <500 followers → launch thread 5k–25k impressions typical; viral is not the plan. Signups track via UTM, not likes.

## Timeline

| Phase | When | Action |

## Tactic stack

1. **`x_icp_account_map` (T-30)** — List 30 accounts (founders, eng leaders, niche VCs). Metric: spreadsheet with last-30d post signal.
2. **`x_profile_bio_hook` (T-30)** — Bio = ICP pain → outcome + link. Pin = best value thread. Metric: bio ≤160 chars, link UTM'd.
3. **`x_build_in_public_cadence` (T-21)** — Mon/Wed/Fri posts. Metric: 12 posts in 28 days minimum.
4. **`x_thread_format_7` (T-14)** — Pre-launch thread drafted. Metric: 7 tweets, tweet 7 = single CTA.
5. **`x_engagement_before_pitch` (T-14)** — 10 thoughtful replies/day, no product links. Metric: 14-day log.
6. **`x_launch_thread_h0` (H0)** — Launch thread published; pinned. Metric: timestamp logged.
7. **`x_reply_sla_2h` (H0–H+8)** — Reply mentions/QRTs ≤2h. Metric: ≥85% SLA.
8. **`x_quote_tweet_launch_ethics` (H0–H+4)** — Max 2 quote-tweets with added insight, never vote asks. Metric: 0 "please RT" language.
9. **`x_cross_post_delay_24h` (T-3)** — LinkedIn launch post ≥24h offset from X thread. Metric: calendar shows gap.
10. **`x_teardown_thread_d1` (D+1)** — Metrics + lessons thread. Metric: published with honest numbers.
11. **`x_no_engagement_pods` (always)** — Zero pod participation. Metric: self-audit signed.

## Orchestration

- **Primary:** Founder X thread + replies
- **Parallel:** community-launch or PH on offset schedule — not same hour
- **Do not:** Engagement pods, mass @ mentions, auto-DMs
- **Support:** Screenshot/GIF assets from launch-asset-generator

## Realistic outcomes

| Profile | Launch thread impressions | Signups (UTM) | New followers (7d) |
|---------|---------------------------|---------------|---------------------|
| Aggressive (2k+ followers) | 25k–120k | 40–300 | 200–800 |
| Standard (300–2k) | 8k–40k | 15–80 | 50–200 |
| New (<300) | 2k–12k | 5–30 | 20–80 |

## Kill / pivot rules

- Launch thread <1k impressions in 4h → check hook tweet; don't delete — reply-chain bump once only
- 0 ICP replies after 24h → rewrite hook; postpone paid boost
- Reply SLA missed >50% → pause new posts; clear notifications backlog before next thread
- Quote-tweet backlash → stop QRTs 72h; teardown with lesson

## Ethics line

- No engagement pods, bought followers, or coordinated RT rings
- Build-in-public uses honest metrics only
- Quote-tweets add context — never vote manipulation across platforms

## Principles

1. **X ≠ LinkedIn.** Hooks are shorter, tone is casual-technical, threads beat single-link drops. Never cross-post LinkedIn essay verbatim — rewrite for X pacing (line breaks, screenshots, numbers).
2. **Thread anatomy:** Tweet 1 = hook + outcome. Tweets 2–5 = problem → build → demo GIF. Tweet 6 = honest limitation. Tweet 7 = single CTA + UTM. Max 7 tweets for launch thread; don't thread-spam 20 tweets.
3. **Build-in-public 3×/week minimum** during runway: Mon = lesson learned, Wed = screenshot/metric, Fri = ask-for-feedback. Not every post needs a CTA — trust compounds.
4. **Reply SLA ≤2 hours** during launch window (H0–H+8). X algorithm rewards fast founder replies in first hour. Block calendar; use notifications, not interns pretending to be founder.
5. **Quote-tweet launch ethics:** Quote your own launch post OR a supporter's honest take — add context ("here's what we shipped"), never "please RT for visibility." Pair with community-launch timing, don't compete for founder attention same hour as Show HN.
6. **Engage before pitch:** Reply thoughtfully on 10 ICP accounts/day for 14 days before first product thread. Cold @ mentions with demo links get muted.
7. **Pinned tweet = best thread**, not landing page ad. Rotate pin at launch week to launch thread; revert to build log after D+7.
8. **8020 mix:** 80% value (tips, teardowns, opinions), 20% product. Deviating to 50% promo triggers unfollows from dev ICP.
9. **UTM on every outbound link:** `utm_source=twitter&utm_medium=organic&utm_campaign=founder_thread`. Track clicks — vanity impressions lie.
10. **Screenshot > link preview** for UI demos. X compresses link cards; a clean screenshot with alt text outperforms.
11. **Teardown thread D+1** with honest metrics (impressions, signups, lessons) — often outperforms launch thread for long-tail follows.
12. **Honest ceiling:** New account <500 followers → expect 2k–15k impressions per good thread, not viral. State ceiling when user asks for "X growth in 1 week."

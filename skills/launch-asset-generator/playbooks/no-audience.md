# Playbook: Launch Asset Generator With No Audience

Launching without an existing audience means assets must **explain and convince cold strangers** — not celebrate with fans. Every piece stands alone; no "you've been following our journey."

- **One positioning spine, enforced across all assets.** Pull from product profile: outcome promise in ≤12 words. Tagline, tweet 1, ad headline, and email subject must share the same verb + noun.
- **PH tagline ≤60 chars — stranger test.** If someone on PH front page can't map it to a category, rewrite. Prefer "X for Y" or outcome verb patterns.
- **Tweet thread: problem → build story → demo → CTA.** 5–7 tweets. Tweet 1 is the hook (pain stat or founder moment). No thread that opens with "Excited to announce." Cold audience needs context first.
- **Maker comment: honest build story + specific ask.** 3 paragraphs: why you built it, one thing that's different (with evidence), ask for feedback from [ICP role]. Upvote ask is implicit on PH — don't beg.
- **Ad primary text: outcome + risk reversal.** "Try free" or "No credit card" for cold traffic. ≤125 chars Meta primary. One variant only at this stage — don't split-test copy you haven't validated organically.
- **No fake social proof.** Zero users = zero "join 10,000 teams." Replace with specificity: "Built by ex-[credible company] engineers" or "Automates [specific workflow] in [time]."
- **Launch email only if you have a list.** No list = skip email asset; focus PH + Twitter + founder LinkedIn post (draft separately in outreach skill if needed).
- **Day-of checklist in every pack.** Post time (PH: Tue–Thu 12:01am PT), supporter ping list (even if empty, note "DM 10 ICP contacts manually"), 2-hour comment reply block, screenshot capture for retrospective.
- **Label everything `DRAFT — DO NOT POST`.** Publishing is always user action. Include character counts on constrained fields.

Track: positioning consistency score (same core phrase in ≥4 assets), constraint compliance (PH ≤60, ad ≤125), asset completeness (tagline + thread + maker comment + 1 ad minimum).

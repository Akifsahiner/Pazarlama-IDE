# Launch Tweet Thread — Fill-In Template

5–7 tweets. **DRAFT — DO NOT POST.** Each tweet ≤280 chars (note count). One positioning spine across all tweets.

---

## Thread config

```
Product:              [name]
Positioning spine:    [outcome promise ≤12 words]
Launch URL:           [PH / landing with UTM]
Launch date:          [YYYY-MM-DD]
Audience assumption:  [cold / warm followers]
Primary CTA:          [try link / PH upvote / reply for access]
```

---

## Tweet map

| # | Role | Content focus |
|---|------|---------------|
| 1 | Hook | Pain stat, founder moment, or contrarian take — NO "excited to announce" |
| 2 | Context | Who it's for + what category |
| 3 | Problem depth | Specific workflow pain (JTBD) |
| 4 | Solution / demo | What it does — show don't tell |
| 5 | Proof | Metric, integration, or trust signal |
| 6 | Social proof or build story | Customer, design partner, or honest "we built this because" |
| 7 | CTA | Link + clear ask |

## Fill-in drafts

### Tweet 1 — Hook
```
[Write hook — pain-first]

Chars: ___/280
```

**Hook patterns:**
- "[Specific pain] costs [ICP] [time/$] every [period]. We got tired of it."
- "I spent [timeframe] doing [manual task]. So we built [product]."
- "Hot take: [incumbent approach] is broken for [ICP]. Here's what we did instead."

### Tweet 2 — Context
```
[Product] is for [ICP role] at [company type] who [JTBD situation].

Chars: ___/280
```

### Tweet 3 — Problem
```
The workflow today:
→ [step 1 painful]
→ [step 2 painful]
→ [step 3 painful]

[One sentence emotional truth from customer/founder research]
```

### Tweet 4 — Solution
```
[Product] [verb] [object]:
• [capability 1 — workflow not feature]
• [capability 2]
• [capability 3]

[Optional: "90-sec demo in replies" if video exists]
```

### Tweet 5 — Proof / trust
```
[B2B: integrations + compliance / Consumer: speed metric]

Examples:
• "Works with Slack, Stripe, and Notion out of the box"
• "SOC 2 Type II — because your security team will ask"
• "Setup in 12 minutes — timed, not estimated"
```

### Tweet 6 — Build story or customer
```
[Honest story — no fake numbers]

"We're live with [N] design partners" OR "[Customer type] cut [metric] from X to Y"
```

### Tweet 7 — CTA
```
[Product] is live [today/on PH/on Product Hunt].

[CTA]: [link with ?utm_source=twitter&utm_medium=organic&utm_campaign=launch]

[Optional: "RT appreciated" only if genuine — never beg]
```

## Bad vs good

| Bad | Good |
|-----|------|
| Tweet 1: "Excited to announce…" | Tweet 1: pain or contrarian hook |
| Feature dump tweet 4 | Workflow verbs: "matches, flags, exports" |
| "Revolutionary AI platform" | Named workflow + ICP |
| Fake "10,000 users" | Honest design partner count or specific metric |
| 12-tweet thread | 5–7 tweets — respect attention |

---

## Pre-post checklist

- [ ] Positioning spine appears verbatim in tweet 2 or 4
- [ ] Every tweet ≤280 chars (counted)
- [ ] Link has UTM parameters
- [ ] No tweet requires prior tweet to understand category
- [ ] Screenshot or demo clip queued for reply to tweet 4
- [ ] Founder blocked 2 hours for replies on launch day

---

## Reject the thread if:

- Opens with announcement without pain context (cold audience)
- >1 tweet is pure feature list without workflow
- Claims not in product profile
- CTA missing or points to generic homepage without UTM
- Character counts exceed 280 on any tweet

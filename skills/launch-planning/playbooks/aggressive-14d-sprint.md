# Playbook: Aggressive 14-day Launch Sprint

Every task maps to a registry tactic with phaseLabel and acceptance criteria. Dependency-aware aggression beats a flat backlog.

## Preconditions
- [ ] Product name + ICP + primary metric locked in MarketingProfile
- [ ] ≥1 distribution asset (list, community, or founder audience)
- [ ] Owner hours: ≥10h/week for 14 days

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | Solo, no list | 8 tasks, learning metrics |
| standard | Warm channel | 10–12 tasks, one primary channel |
| aggressive | Multi-channel ready | 12–15 tasks, parallel tracks, hard kill dates |

## Timeline
- **T-14**: Bottleneck diagnose + event map
- **T-10**: Asset pack + landing CTA lock
- **T-7**: Outbound / content grid start
- **T-3**: Soft audience warm-up
- **H0**: Primary launch motion
- **D+1**: Teardown + next experiment

## Tactic stack
1. **`launch_bottleneck_diagnose`** — Name one bottleneck from profile (not five)
2. **`funnel_event_map`** — Instrument before vanity dashboards
3. **`hero_social_proof_stack`** — Landing ready for the sprint CTA
4. **`launch_task_graph`** — Every plan task: tactic + execution_mode + phaseLabel + acceptance_criteria
5. **`channel_parallel_cap_2`** — ≤2 support channels beside primary
6. **`d1_teardown_metric`** — Ship teardown with numbers, not vibes
7. **`generic_title_ban`** — Reject “improve marketing” / “post on social”

## Orchestration
- Track A (repo/landing) parallel Track B (distribution)
- Browser research tasks ≥2 when playbook needs competitive intel
- Plan Studio instructions_md first line: `Tactic: {id}`

## Realistic outcomes
- Conservative: one shipped motion + baseline metric
- Standard: primary KPI moves into target band
- Aggressive: compound multi-channel — only if assets exist; else honest ceiling

## Kill / pivot rules
- Task without acceptance_criteria → rewrite before run
- Leading metric miss by D+3 → cut secondary channel, double primary
- Generic task titles in plan → hard fail lint

## Ethics line
- No fake metrics in teardown; no ToS-breaking growth hacks

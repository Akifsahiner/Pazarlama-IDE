# Anti-Patterns

- **Don't:** Describe the audience as "early adopters", "tech enthusiasts", or "SMBs". **Why:** Too broad to write copy or run outbound; no qualification filter.
- **Don't:** List features without evidence citations. **Why:** Downstream skills will fabricate claims; trust in the whole stack collapses.
- **Don't:** Invent competitors not in `competitors` input or verifiable public sources. **Why:** Positioning against fake rivals reads as slop to buyers.
- **Don't:** Copy README marketing fluff verbatim as truth. **Why:** READMEs over-promise; cross-check against routes and actual UI flows.
- **Don't:** Skip `claims_needing_validation` when evidence is thin. **Why:** Hiding uncertainty causes outreach and launch assets to assert false metrics.
- **Don't:** Produce a one-liner longer than 12 words. **Why:** Product profiles feed headlines; long one-liners become unusable hero copy.
- **Don't:** Ignore `/pricing` or monetization signals. **Why:** GTM motion (PLG vs sales-led) depends on price point and packaging.
- **Don't:** Conflate framework/stack with customer value. **Why:** "Built with Next.js" is not a differentiator unless the buyer is choosing dev tools.
- **Don't:** Assume B2B because the repo is B2B-shaped. **Why:** Prosumer and indie tools often ship Electron/desktop — verify from copy and onboarding.
- **Don't:** Run shell commands or modify files during intelligence gathering. **Why:** This skill is read-only inspection under `read_inspect` scope.

- **Don't:** Ship generic advice untied to this product profile. **Why:** Founders already have ChatGPT. **Cost:** Trust and retention.

- **Don't:** Recommend five channels at once. **Why:** Dilutes learning. **Cost:** No clear kill signal.

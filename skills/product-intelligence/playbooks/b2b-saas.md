# Playbook: Product Intelligence for B2B SaaS

B2B buyers evaluate risk, integration fit, and ROI — not feature counts. Your product profile must read like a champion's internal memo, not a GitHub README.

- **Lead with the job-to-be-done, not the tech stack.** Read landing hero + `/pricing` + onboarding first. B2B positioning = "[Role] at [company stage] uses [Product] to [measurable outcome] instead of [incumbent/workaround]."
- **Map the buying committee from product surface.** SSO/SAML routes → enterprise buyer. Team invites + roles → manager + admin. API docs + webhooks → technical evaluator. Self-serve signup only → champion-led PLG.
- **Extract integration story from `/integrations`, README, or env examples.** List top 5 connected tools with evidence (route, doc page, or config). Missing integrations list = flag for sales-led deals.
- **Pricing tiers reveal ICP segmentation.** Free/starter → PLG individual champion. Team tier with seat minimum → department buyer. Enterprise "Contact us" → ACV >$10k, security review required. Document each tier's implied buyer.
- **Security/compliance signals from routes and docs.** SOC 2 page, GDPR mention, audit logs, data residency — each unlocks enterprise positioning. Absence = "not enterprise-ready" in profile.
- **Competitive wedge on one axis only.** vs. Salesforce: faster setup. vs. spreadsheet: fewer errors. vs. internal build: lower TCO. Never claim "better on everything."
- **Quantify outcomes from case studies, testimonials, or README metrics.** "Saves 6 hours/week" beats "automates workflows." If no numbers exist, mark outcome fields as `claims_needing_validation`.
- **Document activation path as the sales demo script.** Signup → connect integration → first value action → team invite. Each step = a route. This becomes outreach proof and launch asset fodder.
- **Flag PLG vs sales-led motion explicitly.** Single CTA "Start free" = PLG. Dual CTA "Book demo" + "Start trial" = hybrid. Demo-only = sales-led. Downstream outreach and launch assets depend on this.

Track: claim_evidence_ratio, buying-committee coverage (champion + evaluator + economic buyer addressed in copy), integration count with sources.

## Preconditions
- [ ] Marketing profile complete for product-intelligence
- [ ] Primary metric named before execution
- [ ] Honest aggression dial matches real assets (list, proof, team hours)

## Aggression dial

| Level | When | Focus |
|-------|------|-------|
| conservative | Thin assets / solo | One channel, learning metrics |
| standard | Warm channel or list | Best-practice cadence from narrative above |
| aggressive | Strong proof + distribution | Execute full tactic stack with kill rules |

## Timeline
- **T-14**: Instrument leading metric; lock ICP + offer
- **T-7**: Ship assets referenced in narrative
- **T-1**: Final checklist + supporter warm-up
- **H0**: Primary launch motion
- **D+1…D+7**: Measure, teardown, kill or scale

## Tactic stack
1. Name bottleneck with 3 profile citations
2. Ship one measurable experiment this week
3. Tie Plan Studio tasks to registry tactic ids
4. Cap parallel channels at 2
5. Predefine kill/pivot rule before spend or blast

## Orchestration
- One primary channel from narrative; ≤2 support channels
- Repo/browser/asset tasks split by execution_mode

## Realistic outcomes
- State honest bands — do not promise #1 PH or viral without assets
- Leading metric moves in 7–14d when playbook matches profile

## Kill / pivot rules
- Leading metric misses after agreed sample → stop and rewrite hypothesis
- Never add channels when primary motion underperforms

## Ethics line
- No fake metrics, fake social proof, undisclosed ads, or ToS-breaking growth hacks

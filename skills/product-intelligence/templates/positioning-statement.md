# Positioning Statement — Fill-In Template

One positioning spine for all GTM copy. Strangers should repeat it after one read.

---

## Core formula

```
For [ICP role] at [company type/stage]
who [trigger / situation],
[Product name] is the [category]
that [primary outcome / measurable result]
unlike [incumbent or workaround],
because [single differentiated wedge — one axis only].
```

## Wedge axis picker (choose ONE)

Check the single axis you win on — not all of them.

- [ ] **Speed** — faster to value, faster workflow, faster setup
- [ ] **Cost** — lower TCO vs incumbent or headcount
- [ ] **Simplicity** — fewer steps, no admin, no config tax
- [ ] **Integration** — native fit with stack they already use
- [ ] **Compliance / trust** — audit-ready, secure, governed
- [ ] **Quality of output** — fewer errors, higher conversion, better data

## Examples

**Good — B2B SaaS:**
```
For finance controllers at Series A–C SaaS companies
who need to close books before board meetings,
Closeflow is a month-end automation tool
that cuts close time from 10 days to 2
unlike NetSuite's manual reconciliation modules,
because it auto-matches Stripe + bank feeds with exception-only review.
```

**Good — PLG tool:**
```
For indie founders shipping side projects
who lose weekends to deploy config,
Pieter is a deployment platform
that goes from git push to production in under 3 minutes
unlike wrangling AWS Console + Vercel + Sentry separately,
because it's one pipeline with errors and revenue built in.
```

**Bad — reject and rewrite:**
```
For businesses of all sizes who want to grow,
GrowthAI is a next-gen AI-powered platform
that revolutionizes marketing with seamless automation
unlike legacy tools,
because we use cutting-edge AI.
```
*Fails: no ICP, no outcome, no real incumbent, hype words, no evidence.*

## Validation test

Read aloud to someone outside the company. They should answer:
1. Who is it for?
2. What does it do?
3. Why not the alternative?

If any answer is wrong — rewrite the statement, not the explanation.

---

## Reject if:

- Uses: revolutionary, seamless, intuitive, AI-powered (without specific capability), platform, solution
- Names 2+ competitors in the "unlike" clause
- Outcome is not measurable or observable
- "For everyone" or missing company stage/role
